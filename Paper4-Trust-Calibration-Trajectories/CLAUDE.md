### Key Terminology
- **R_b:** Behavioral Reliance (adaptive_offer proportion)
- **P:** Performance (accuracy)
- **P_adaptive:** Performance on AI-recommended questions only
- **Gap:** R_b - P (negative = under-reliance, positive = over-reliance)
- **ABE:** AI Benefit Emergence (5th trajectory pattern — P_adaptive rises, R_b flat)
- **MBC:** Model-Based Clustering (mclust, wide-format)
- **LCGA:** Latent Class Growth Analysis (lcmm, growth curves)

### Analysis Phases
- Phase 0: Data diagnostics (filtering criteria, zero-inflation)
- Phase 1: Time series construction (R_b, P per window)
- Phase 1B: Re-operationalization (P_adaptive, P_non_adaptive)
- Phase 2A: MBC via mclust (6-class VEE)
- Phase 2B: MBC with alternative operationalization (8-class VVV)
- Phase 2C: LCGA via lcmm (4-class)
- Phase 3: RQ3 (prediction) + RQ4→RQ5 (outcomes)
- Phase 4: Visualization + Sensitivity analysis
